### 03-game-state-events-decisions.s005 (Event journal)
Every atomic mutation emits events. Trigger detection consumes events, not actions.

```ts
interface EngineEvent {
  id: EngineEventId;
  seq: number;
  type: EngineEventType;
  actor?: PlayerId;
  source?: CardRef;
  affected?: CardRef[];

  visibility: EventVisibility;
  createdAtStateSeq: StateSeq;
```

### Event visibility

### 03-game-state-events-decisions.s006 (Event visibility)
Events may contain hidden data. Filter them before sending to clients.

```ts
type EventVisibility =
  | { type: 'public' }
  | { type: 'private'; playerId: PlayerId }
  | { type: 'hidden' }
  | { type: 'replayOnly' };
```

## Atomic mutation contract

### 03-game-state-events-decisions.s007 (Atomic mutation contract)
Every primitive state mutation uses the same return shape.

```ts
interface EngineStepResult {
  state: GameState;
  events: EngineEvent[];
}

type AtomicMutation = (state: GameState) => EngineStepResult;
```

The engine should not mutate state in place in production logic. Dev/test may use deep-freeze to catch accidental mutation.

## Causality

### 03-game-state-events-decisions.s008 (Causality)
Use causality references to make replays and debugging readable.

```ts
type CausalityRef =
  | { type: 'playerAction'; actionId: string }
  | { type: 'effect'; queueEntryId: string; effectId: string }
  | { type: 'ruleProcess'; name: string }
  | { type: 'replacement'; replacementId: string }
  | { type: 'decision'; decisionId: string };
```

## Pending decisions

### 03-game-state-events-decisions.s018 (Canonical event visibility)
Each `EngineEvent` has one visibility policy:

```text
public          safe for both players immediately
private         visible only to listed player IDs
replayOnly      hidden during live play but available in completed full replay
serverOnly      never leaves trusted server/runtime logs
```

Visibility is independent of replay determinism. Replay artifacts may store information that was never sent to either player during the live match.

## Deterministic RNG

## Scope
- Create an engine event factory that assigns EngineEventId, seq (event-local), createdAtStateSeq, and visibility.
- Enforce 'every atomic mutation emits events' by requiring AtomicMutation results to include events and appending them to state.eventJournal in a single place.
- Add causality references (playerAction/effect/ruleProcess/replacement/decision) plumbing so events can be tagged with causedBy by callers.
- Implement a minimal filterEventsForPlayer(events, playerId) utility based on EventVisibility (public/private/hidden/replayOnly) without leaking hidden payloads.

## Out of Scope
- Full PlayerView state filtering (handled elsewhere).
- Replay writer/storage format.
- Card/effect trigger detection logic (only event plumbing).

## Constraints
- Do not place transport envelopes in canonical EngineEvent payloads.
- Events must remain JSON-serializable.

## Required Tests
- Unit: filterEventsForPlayer includes/excludes each visibility type correctly.
- Integration: applying a simple action results in at least one event in result.events and appended to state.eventJournal with createdAtStateSeq matching result.state.seq.
