## Milestone 7 - public unranked alpha and custom lobbies

## Scope
- Implement canonical JSON serializer with stable object-key ordering and stable array ordering (do not reorder semantic arrays).
- Implement hashState({state, includeHidden, normalizeTransientIds}) producing a stable string hash.
- Ensure timestamps are excluded unless explicitly part of replay logic (do not include wall-clock times in canonical state hashing).
- Wire EngineResult.stateHash to hashState(includeHidden:true, normalizeTransientIds:false) for authoritative use in tests/CLI.

## Out of Scope
- Replay checkpointing/storage.
- Client-sync public hash unless needed later (optional output not required).

## Constraints
- Do not use nondeterministic JSON stringify output without key sorting for hashing.
- Do not incorporate process-unique data (timestamps, random UUIDs) into state hashing inputs.

## Required Tests
- Integration: run a short deterministic sequence twice and assert final stateHash matches.
- Unit: canonical JSON serialization produces identical output for objects with differing key insertion orders.

## Expected Output
- code changes

## Acceptance Criteria
- [ ] Same seed + same action log yields identical final stateHash across runs (M1-015).
- [ ] Hashing function is stable across node runs/platforms for the same input JSON structure.
- [ ] Hashing includes hidden data when includeHidden is true.
