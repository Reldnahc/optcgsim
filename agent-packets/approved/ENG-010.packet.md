## Why
Implement phase progression (refresh/draw/don/main/end) with first-turn restrictions: first player skips first draw, DON!! placement limits, and no attacks on either player's first turn.

### 02-engine-mechanics.s037 (First-turn restrictions)
The engine must track first/second player and each player's first turn.

| Player / turn | Draw Phase | DON!! Phase | Attack |
|---|---:|---:|---:|
| Player going first, first turn | No draw | Place only 1 DON!! | Cannot attack |
| Player going second, first turn | Draw normally | Place 2 DON!! if available | Cannot attack |

## Milestone 2 - first effect runtime

## Scope
- Implement TurnState tracking: globalTurn, playerTurnCounts, turnPlayerId, phase, and battle step placeholder.
- Implement phase transitions and emit phaseStarted/phaseEnded events.
- Implement Draw Phase behavior including first-player first turn 'no draw'.
- Implement DON!! Phase behavior including placing 1 DON!! for first player's first turn, 2 DON!! for second player's first turn if available (from DON!! deck).
- Implement baseline 'cannot attack' restriction for both players on their first turn; wire into legal actions for declareAttack.

## Out of Scope
- Detailed battle step sub-structure beyond gating attack legality.
- Card-specific effects that modify turn structure.

## Constraints
- Phase progression must be deterministic and driven by state (no wall clock).

## Required Tests
- Integration: simulate two turns and assert draw counts and DON!! placements match first-turn restriction table.
- Integration: getLegalActions does not include declareAttack on either player’s first turn.
